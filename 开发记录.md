# 开发记录

## 关于 DI 框架

- 由于使用的数据源为校方闭源，不可控，故使用 DI 框架，方便进行测试。
- 便于热更新
- 为统一风格，单例模式也使用 DI 实现

## 关于项目架构 分层MVVM with 防腐层

### 数据源模型

- 本地数据源，即缓存
- 远程数据源，即接口，仅负责网络请求
- 防腐层，仅负责将网络json转换成内部模型类实例
- 仓库类，调用数据源、远程数据源、防腐层。是程序中访问所有数据的入口。
  仓库层将底层抛出的异常封装成 Either，将null值也封装成Left
- 领域层，
- UI 层，不应该在UI层处理任何null，应该在UI中使用fold处理either

## 关于目录结构

### providers 文件夹的位置

- 在 datasource、repository、anti_corruption 等组件目录下分别创建 providers 子文件夹，存放对应的 provider 文件。

- 保持每个组件内部高内聚，便于模块独立开发、测试和维护。

- 如果某个组件特别简单，允许直接在组件目录下放置 xxx_provider.dart 文件（不建 providers 子目录），但项目应统一风格。
